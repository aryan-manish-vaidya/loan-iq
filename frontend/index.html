Synthetic Dataset Generation
How each feature and target is generated — distributions, formulas, and correlations
5,000 samples · 9 features · 2 targets
Feature / Target Role Distribution / Formula Range Key Correlations
age Input Normal(μ=38, σ=10) · clipped [18, 70] 18 – 70 Mild ↑ income
income Input LogNormal(μ=11.0, σ=0.6) · clipped [$15K, $250K] $15K – $250K Base for DTI & LTI ratios
employment_status Input Categorical: Full-time 60% · Self-empl 20% · Part-time 12% · Unemployed 8% 0 – 3 ↓ interest rate (bonus −2%)
credit_score Input Normal(μ=650, σ=80) · clipped [300, 850] (FICO scale) 300 – 850 Weight 0.30 in approval · up to +8% on rate
loan_amount Input LogNormal(μ=10.0, σ=0.8) · clipped [$1K, $100K] $1K – $100K Used in LTI ratio → approval & rate
loan_term Input Categorical: 36m 30% · 48m 20% · 60m 20% · other 30% 12 – 120 mo Independent feature
existing_debts Input LogNormal(μ=9.0, σ=1.0) · clipped [$0, $80K] $0 – $80K Weight 0.25 in approval via DTI
payment_history Input Normal(μ=70, σ=18) · clipped [0, 100] 0 – 100 Weight 0.20 in approval · ↑ missed_emis
missed_emis Input Poisson(λ = (1 − payment_history/100) × 4) · clipped [0, 12] 0 – 12 Correlated with payment_history; +0.6%/missed on rate
debt_to_income Derived existing_debts / (income + 1) 0 – 1 (clipped) Used in approval score & rate formula
loan_approved Target score = 0.30·FICO + 0.25·(1−DTI) + 0.20·pay_hist + 0.15·employ + 0.10·(1−LTI) − 0.08·missed + noise → 1 if score > 0.72 0 / 1 ~60% approval rate
interest_rate Target 8.0 + (1−FICO)·8 + missed·0.6 + (1−pay_hist)·4 + DTI·3 − employ·2 + LTI·1.5 + noise(σ=0.6) 6% – 20% Avg ≈11.5% · std ≈3.2%
ℹ️ All continuous features are generated with controlled correlations to simulate realistic credit behaviour. Gaussian noise (σ=0.05) is added to the approval score to prevent perfect separability, ensuring the Random Forest learns probabilistic boundaries rather than exact rules.
